# Maintainer Playbook

This guide helps keep Open Source Contribution Atelier beginner-friendly and contributor-ready.

## Weekly Cadence

1. Review all issues with `needs-triage`.
2. Archive or close stale/duplicate issues.
3. Pick at least 3 `good first issue` tasks with clear acceptance criteria.
4. Review pending help requests from learners.

## Issue Lifecycle

1. Opened: author provides problem statement and reproduction.
2. Triaged: labels + owner + acceptance criteria added.
3. In progress: linked PR or assigned contributor.
4. Resolved: merged and closed with release note reference.

## Label Taxonomy

- `bug`: Broken behavior
- `enhancement`: Product improvement
- `curriculum`: Lesson or exercise changes
- `good first issue`: Suitable for first-time contributors
- `needs-triage`: Not yet reviewed by maintainers
- `blocked`: Waiting on external dependency or decision

## PR Review Rules

- Do not merge PRs without test evidence for changed behavior.
- Require screenshots for frontend UX changes.
- Keep PR scope tight; request split PRs when needed.
- Prefer actionable comments over broad criticism.

## Curriculum Quality Checklist

Every lesson should include:

- clear audience level
- 2-5 learning objectives
- at least one exercise with expected command or expected result
- tips/common pitfalls
- estimated completion time
